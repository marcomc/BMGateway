## Propositions

- [ ] Add camera-based barcode scanning for device MAC or serial capture.
  Manual MAC entry is error-prone on phones and laptops. The original mobile
  app supports scanning the device barcode, and the web UI should offer the
  same shortcut when camera access is available.
  Actions:
  - add a scan action to the add-device flow next to the MAC or serial input
  - support camera access on desktop and mobile browsers when permitted
  - decode the scanned barcode and normalize the captured value into the
    expected MAC-address format used by the registry
  - keep manual entry as the fallback path when the camera is unavailable or
    the scan fails
  - define validation and error messaging for unreadable or unsupported barcode
    payloads

- [ ] Add Raspberry Pi USB-OTG image-export mode for external monitors and
  digital picture frames.
  The gateway should be able to present itself through the Raspberry Pi Zero
  USB-OTG hardware path as a thumb-drive-like mass-storage device whose image
  files are periodically regenerated by `BMGateway`. A monitor or digital photo
  frame such as the Samsung `SPF-71E` could then rotate through those images to
  show Fleet Trend and other gateway views without a browser.
  Actions:
  - investigate the Raspberry Pi Zero USB Plug / USB-OTG hardware path and the
    Linux gadget configuration needed to expose a mass-storage device
  - define a generated-image set with stable filenames so the attached display
    can rotate through them repeatedly
  - generate Fleet Trend and other summary images on a periodic schedule
  - update those images in place so the picture frame reloads fresh data while
    keeping the same file names
  - define the runtime/config model for image cadence, enabled views, and
    output resolution/profile per target display
  - validate the behavior against the Samsung `SPF-71E` and document the setup
